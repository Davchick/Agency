# Спецификация сайта агентства интернет-рекламы
## ИП Ледовских Ксения Андреевна

---

## 1. Обзор проекта

**Тип:** Многостраничный корпоративный сайт с личным кабинетом  
**Стек:** React (Vite) + TailwindCSS + React Router  
**Дизайн:** Минималистичный, принципы Apple — много воздуха, чистые линии, сдержанная палитра, акцент на контенте  
**Цель:** Представление услуг агентства, привлечение клиентов, сбор заявок, оформление заказов через личный кабинет

---

## 2. Дизайн-система

### 2.1 Цветовая палитра

Основана на нейтральных тонах с одним акцентным цветом:

| Роль | Цвет | Hex |
|------|------|-----|
| Primary (фон основной) | Белый | `#FFFFFF` |
| Secondary (фон альтернативный) | Светло-серый | `#F5F5F7` |
| Text Primary | Тёмно-серый | `#1D1D1F` |
| Text Secondary | Серый | `#6E6E73` |
| Accent | Синий (Apple-style) | `#0071E3` |
| Accent Hover | Тёмно-синий | `#0077ED` |
| Border | Светло-серый | `#D2D2D7` |
| Success | Зелёный | `#34C759` |
| Error | Красный | `#FF3B30` |

### 2.2 Типографика

Системные шрифты (как у Apple) — быстрая загрузка, нативный вид:

```
font-family: -apple-system, BlinkMacSystemFont, 'SF Pro Display', 'Segoe UI', Roboto, sans-serif
```

| Стиль | Размер | Вес | Line-height |
|-------|--------|-----|-------------|
| Hero Heading | 56-72px | 700 | 1.05 |
| Section Heading | 40-48px | 600 | 1.1 |
| Subheading | 24-28px | 500 | 1.2 |
| Body Large | 19px | 400 | 1.5 |
| Body | 17px | 400 | 1.5 |
| Small | 14px | 400 | 1.4 |
| Caption | 12px | 400 | 1.3 |

### 2.3 Отступы и сетка

- **Container max-width:** 1200px (центрованный)
- **Section padding:** 120px вертикально (desktop), 80px (tablet), 60px (mobile)
- **Spacing scale (Tailwind):** 4, 8, 12, 16, 24, 32, 48, 64, 96, 128
- **Grid:** 12 колонок, gutter 24px, margin 48px

### 2.4 Компоненты UI

**Кнопки:**
- Primary: синий фон, белый текст, border-radius 980px (pill), padding 16px 32px
- Secondary: прозрачный фон, синий текст, синяя обводка 1px
- Минимальный hover: лёгкое затемнение + scale 1.02

**Карточки:**
- Белый фон, border-radius 20px, тень subtle (0 4px 24px rgba(0,0,0,0.06))
- Hover: тень усиливается, лёгкий подъём (translateY -4px)

**Формы:**
- Поля ввода: border-radius 12px, обводка 1px #D2D2D7, фокус — синий outline
- Label: 14px, #6E6E73, margin-bottom 8px

### 3.2 Навигация

**Header (fixed, sticky):**
- Логотип (слева) — текстовый или SVG
- Меню (центр): Главная | О нас | Услуги | Кейсы | Контакты
- Справа:
  - Не авторизован: кнопка «Войти» → `/login`
  - Авторизован: аватар/имя → dropdown (Мой кабинет | Мои заказы | Профиль | Выйти)
- CTA-кнопка (справа, до авторизации): «Оставить заявку» → `/contact`
- Мобильное меню: hamburger → fullscreen overlay

**Footer:**
- Колонка 1: Логотип + краткое описание
- Колонка 2: Навигация (ссылки на страницы)
- Колонка 3: Услуги (ссылки)
- Колонка 4: Контакты (телефон, email, соцсети)
- Низ: «© 2026 ИП Ледовских Ксения Андреевна» | Политика конфиденциальности | Условия использования

## 6. Функциональные требования

### 6.1 Навигация
- [ ] React Router v6 для маршрутизации
- [ ] Scroll to top при смене страницы
- [ ] Active state в меню для текущей страницы
- [ ] Мобильное меню с анимацией (slide/fade)

### 6.2 Анимации
- [ ] Fade-in секций при скролле (Intersection Observer)
- [ ] Hover-эффекты на карточках и кнопках
- [ ] Плавный скролл к якорям
- [ ] Анимация появления элементов в списке (stagger)

### 6.3 Формы
- [ ] Валидация полей (обязательные, email, телефон)
- [ ] Состояния: loading, success, error
- [ ] Feedback после отправки (toast или inline-сообщение)
- [ ] Placeholder для интеграции с бэкендом/API

### 6.4 Адаптивность
- [ ] Mobile-first подход
- [ ] Breakpoints: sm(640), md(768), lg(1024), xl(1280)
- [ ] Корректное отображение на 320px+
- [ ] Touch-friendly элементы (минимум 44px для кнопок)

### 6.5 SEO
- [ ] React Helmet для мета-тегов
- [ ] Семантическая разметка (header, main, section, article, footer)
- [ ] Open Graph теги для соцсетей
- [ ] Структурированные данные (JSON-LD) для организации

### 6.6 Производительность
- [ ] Lazy loading изображений
- [ ] Code splitting по страницам
- [ ] Оптимизированные SVG-иконки
- [ ] Минимальный bundle size

### 6.7 Авторизация и заказы
- [ ] Auth Context для глобального состояния (isAuthenticated, user, login, logout)
- [ ] Protected Route — редирект на `/login` для закрытых страниц
- [ ] Persist auth-состояния в localStorage
- [ ] Форма входа с валидацией
- [ ] Форма регистрации с валидацией (email, совпадение паролей, согласие)
- [ ] Оформление заказа в 3 шага (услуга → бриф → подтверждение)
- [ ] Отображение статусов заказа (стейджи + бейджи)
- [ ] Placeholder для API (mock-данные заказов)
- [ ] Logout с очисткой состояния
- [ ] DashboardLayout с боковой навигацией

### 6.8 Юридические требования (152-ФЗ)
- [ ] Чекбокс согласия на обработку персональных данных (регистрация, формы)
- [ ] Ссылка на политику конфиденциальности во всех формах
- [ ] Политика конфиденциальности — отдельная страница с полным текстом
- [ ] Условия использования — отдельная страница
- [ ] Реквизиты ИП в футере /terms
- [ ] Cookie-баннер при первом посещении (placeholder)
- [ ] Возможность отзыва согласия (через `/contact` или удаление аккаунта)

## 10. Бэкенд и API

**На текущий момент бэкенда нет.** Проект — фронтенд-only. Все данные работают на mock-уровне:
- Авторизация хранится в `localStorage` (без серверной валидации)
- Заказы — mock-данные из `src/data/orders.js`
- Формы заявок (контакты, новые заказы) — имитация отправки с success-экраном
- API-запросы пока некуда направлять — все функции-заглушки в `src/utils/api.js`

**Это нормально.** Фронтенд спроектирован так, что при появлении бэкенда достаточно заменить mock-функции на реальные API-вызовы в `src/utils/api.js`.
